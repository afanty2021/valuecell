## 简介

ValueCell是一个社区驱动的金融应用多智能体平台，旨在构建全球最大的去中心化金融智能体社区。该平台提供一组顶级投资智能体，帮助用户进行股票选择、研究、跟踪甚至交易。系统将所有敏感信息存储在用户的本地设备上，确保核心数据安全。

本快速入门指南为新用户提供清晰的上手步骤，涵盖下载和安装桌面应用、配置模型提供商和交易所API凭据、创建交易策略以及监控和控制交易的完整流程。同时为新用户和开发者提供不同的指导路径。

## 新用户指南

对于新用户，可以通过以下步骤快速开始使用ValueCell：

### 下载和安装

要快速开始，请从GitHub的[发布页面](https://github.com/ValueCell-ai/valuecell/releases)下载适用于MacOS或Windows的最新ValueCell应用程序。您也可以从我们的官方网站下载：[https://valuecell.ai](https://valuecell.ai)。

安装完成后，首次使用ValueCell前请配置您首选的模型提供商。请参考应用程序或文档中的说明。

### 实盘交易流程

1. **配置AI模型**：通过Web界面添加您的AI模型API密钥
2. **配置交易所**：设置Binance/HyperLiquid/OKX/Coinbase等API凭据
3. **创建策略**：结合AI模型与交易所创建自定义策略
4. **监控与控制**：实时启动/停止交易员并监控性能

**注意**：目前仅支持合约交易（现货交易以1X合约形式实现），请确保您的合约账户有足够的余额。

## 配置模型提供商

ValueCell支持多种LLM提供商，包括OpenRouter、SiliconFlow、Azure、Openai-compatible、Google、OpenAI和DeepSeek。

### 获取API密钥

选择至少一个模型提供商并获取API密钥：

| 提供商 | 注册链接 |
| --- | --- |
| **OpenRouter** | [openrouter.ai](https://openrouter.ai/) |
| **SiliconFlow** | [siliconflow.cn](https://www.siliconflow.cn/) |
| **Google** | [ai.google.dev](https://ai.google.dev/) |
| **OpenAI** | [platform.openai.com](https://platform.openai.com/) |
| **DashScope** | [bailian.console.aliyun.com](https://bailian.console.aliyun.com/#/home) |

### 配置环境变量

1. 复制示例文件：
```bash
cp .env.example .env
```

2. 编辑`.env`文件并添加您的API密钥：

```bash
# OpenRouter
OPENROUTER_API_KEY=your_openrouter_api_key

# SiliconFlow
SILICONFLOW_API_KEY=your_siliconflow_api_key

# Google
GOOGLE_API_KEY=your_google_api_key

# OpenAI
OPENAI_API_KEY=your_openai_api_key
```

3. （可选）设置主要提供商：
```bash
PRIMARY_PROVIDER=openrouter
```

## 创建交易策略

通过ValueCell的界面创建交易策略，结合AI模型与交易所进行智能交易。

### 策略类型

ValueCell支持多种交易策略类型：

1. **基于提示的策略 (PromptBasedStrategy)**：使用AI模型根据预设提示进行交易决策
2. **网格策略 (GridStrategy)**：在预设价格区间内自动执行买入和卖出

### 创建策略步骤

1. 在Web界面中导航到策略创建页面
2. 选择策略类型
3. 输入策略名称
4. 设置初始资本
5. 配置最大杠杆（1-5倍）
6. 选择交易符号（如BTC/USDT）
7. 设置决策间隔（10-3600秒）
8. 对于基于提示的策略，选择或创建提示模板

### 配置验证

根据`frontend/src/constants/schema.ts`中的验证规则：

```typescript
export const tradingStrategySchema = z.object({
  strategy_type: z.enum(["PromptBasedStrategy", "GridStrategy"]),
  strategy_name: z.string().min(1, "策略名称是必需的"),
  initial_capital: z.number().min(1, "初始资本至少为1"),
  max_leverage: z
    .number()
    .min(1, "杠杆至少为1")
    .max(5, "杠杆最多为5"),
  symbols: z.array(z.string()).min(1, "至少需要一个符号"),
  template_id: z.string().min(1, "需要选择模板"),
  decide_interval: z
    .number()
    .min(10, "间隔至少为10秒")
    .max(3600, "间隔最多为3600秒"),
});
```

## 监控和控制交易

一旦创建了交易策略，您可以通过Web界面实时监控和控制交易活动。

### 监控功能

- **实时性能监控**：查看策略的实时盈亏、交易历史和执行状态
- **交易日志**：查看详细的交易执行日志和AI决策过程
- **风险指标**：监控杠杆使用情况、保证金水平和潜在风险

### 控制功能

- **启动/停止交易**：随时启动或停止策略的交易活动
- **参数调整**：在运行时调整策略参数（如杠杆、交易对等）
- **紧急停止**：在市场异常情况下立即停止所有交易

### 访问监控界面

1. 启动应用程序后，访问 [http://localhost:1420](http://localhost:1420)
2. 导航到"策略"或"交易"页面
3. 选择您要监控的策略
4. 查看实时数据和控制选项

## 常见问题解答

### 如何重置本地数据？

如果遇到数据库兼容性错误，可以删除以下位置的本地数据：

- **LanceDB目录**（系统应用程序目录，与`.env`相同路径）：
  - macOS: `~/Library/Application Support/ValueCell/lancedb`
  - Linux: `~/.config/valuecell/lancedb`
  - Windows: `%APPDATA%\\ValueCell\\lancedb`
- **知识目录**（系统应用程序目录，与`.env`相同路径）：
  - macOS: `~/Library/Application Support/ValueCell/.knowledge`
  - Linux: `~/.config/valuecell/.knowledge`
  - Windows: `%APPDATA%\\ValueCell\\.knowledge`
- **SQLite数据库文件**（系统应用程序目录，与`.env`相同路径）：
  - macOS: `~/Library/Application Support/ValueCell/valuecell.db`
  - Linux: `~/.config/valuecell/valuecell.db`
  - Windows: `%APPDATA%\\ValueCell\\valuecell.db`

### 如何配置OKX交易？

添加OKX凭据到`.env`文件（或在启动前导出）：

```bash
AUTO_TRADING_EXCHANGE=okx
OKX_NETWORK=paper          # 验证后切换到主网
OKX_API_KEY=...
OKX_API_SECRET=...
OKX_API_PASSPHRASE=...
OKX_ALLOW_LIVE_TRADING=false
OKX_MARGIN_MODE=cash       # 或 cross / isolated
```

使用`./start.sh --exchange okx --network paper`启动，通过OKX路由自动交易智能体。只有在准备进行真实执行时，才将`OKX_ALLOW_LIVE_TRADING`设置为`true`。

**重要提示**：在OKX模拟环境中验证策略之前，请保持`OKX_ALLOW_LIVE_TRADING=false`。将API密钥视为生产凭据，并将其存储在安全的保险库中。

### 如何添加新的模型提供商？

1. 在`configs/providers/`目录中创建提供商YAML文件
2. 在`valuecell/adapters/models/factory.py`中实现提供商类
3. 在`ModelFactory._providers`中注册提供商
4. 在`config.yaml`提供商注册表中添加
5. 为提供商配置添加测试
